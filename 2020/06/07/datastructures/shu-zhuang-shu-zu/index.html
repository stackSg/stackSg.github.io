树状数组
数据结构 树状数组
数据结构
发布日期:   2020-06-07
更新日期:   2020-06-09
阅读次数:  

定义

  • 树状数组或二元索引树（英语：Binary Indexed Tree），又以其发明者命名为Fenwick树。多用于高效计算数列的前缀和， 区间和。它可以以O(logn)的时间得到任意前缀和，并同时支持在O(logn)时间内支持动态单点值的修改。
  • BIT是能够完成下述操作的数据结构：
    给一个初始值全为0的数列a(1),a(2),…,a(n)
    1. 给定i,计算a(1)+a(2)+…+a(i)
    2. 给定i和x,执行a(i)+=x
  • 前缀和同样可以使用线段树实现，但是BIT实现起来更方便。

BIT的更新

  • 将元素转换为树形结构如下如：

  • 使第i项的值增加x需要从i开始，不断把当前位置的值增加x，并把i的二进制最低非0位对应的数字加到i上，直到i的值为节点总数。
  • i&(-i)或i&(i-1)能够得到i的最低非0位对应的数字。
  • 创建树状数组时只要给出初始值为0的数组，然后通过更新操作不断加入每个节点值即可建立树状数组。

BIT的求和

  • 计算前i项的和需要从i开始，不断把当前位置i的值加到结果中，并从中减去的二进制最低非0位
    对应的数字，直到变成0为止。

实现

public class BIT {
    //树状数组
    int sums[];
    public BIT(int n){
        //树状数组下标从1开始
        sums = new int[n+1];
    }
    /**
     * 更新sums[i]+=delta
     * @param i
     * @param delta 变化量
     */
    public void update(int i, int delta){
        while (i < sums.length){
            sums[i] += delta;
            i += lowbit(i);
        }
    }

    public int query(int i){
        int sum = 0;
        while (i > 0){
            sum += sums[i];
            i -= lowbit(i);
        }
        return sum;
    }

    private static int lowbit(int i){
        return i & (-i);    
    }
}

应用

区域和检索

  • leetcode 307 区域和检索
public class NumArray {
    private BIT bit;
    private int[] nums;
    public NumArray(int[] nums) {
        this.nums = nums;
        bit = new BIT(nums.length);
        for (int i = 0; i < nums.length; i++) {
            bit.update(i + 1, nums[i]);
        }
    }

    public void update(int i, int val) {
        bit.update(i + 1, val - nums[i]);
        nums[i] = val;
    }

    public int sumRange(int i, int j) {
        return bit.query(j + 1) - bit.query(i);
    }

    public static void main(String[] args) {
        NumArray l = new NumArray(new int[]{1, 3, 5});
        System.out.println(l.sumRange(0, 2));
        l.update(1, 2);
        System.out.println(l.sumRange(0, 2));
    }
}

计算右侧小于当前元素的个数

  • leetcode 315. 计算右侧小于当前元素的个数

  • 暴力破解需要逐个扫描后续元素，复杂度为O(n(2))，考虑优化。

  • 计算数列排名，统计排名的出现次数，然后根据这个频率数组判断元素之前的小于自己的元素个数。

  • 统计profix sum正好使用树状数组，但题目要求为统计右侧，反转数组后可转移到统计左侧。

  • 优化后整体复杂度降为O(logn)

public class Solution {
    public List<Integer> countSmaller(int[] nums) {
        //1.reverse
        List<Integer> list = new ArrayList<Integer>();
        for (int i = 0; i < nums.length; i++) {
            list.add(nums[i]);
        }
        Collections.reverse(list);
        //2.unique and sort
        TreeSet<Integer> set = new TreeSet<>(list);
        Integer[] sorted = new Integer[set.size()]; 
        set.toArray(sorted);
        //3.rank
        Map<Integer, Integer> ranks = new HashMap<>();
        int rank = 0;
        for (int i = 0; i < set.size(); i++) {
            ranks.put(sorted[i], ++rank);
        }
        //BIT
        BIT bit = new BIT(sorted.length);
        List<Integer> res = new ArrayList<>();
        for (int i = 0; i < list.size(); i++) {
            Integer r = ranks.get(list.get(i));
            res.add(bit.query(r-1));
            bit.update(r, 1);
        }
        Collections.reverse(res);
        return res;
    }

    public static void main(String[] args) {
        Solution s = new Solution();
        System.out.println(s.countSmaller(new int[]{7,1,3,2,9,2,1}));
    }
}
