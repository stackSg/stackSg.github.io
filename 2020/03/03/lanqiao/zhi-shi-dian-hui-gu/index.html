  4. binarySearch(Object[], Object key)

    a: 要搜索的数组

    key：要搜索的值

    如果key在数组中，则返回搜索值的索引；否则返回-1或“-”（插入点）。插入点是索引键将要插入数组的那一点，即第一个大于该键的元素的索引。

    技巧：

    [1] 搜索值不是数组元素，且在数组范围内，从1开始计数，得“ - 插入点索引值”；

    [2] 搜索值是数组元素，从0开始计数，得搜索值的索引值；

    [3] 搜索值不是数组元素，且大于数组内元素，索引值为 – (length + 1);

    [4] 搜索值不是数组元素，且小于数组内元素，索引值为 – 1。

  3. 阶梯Nim博弈问题中是将相邻两个棋子之间的间隔抽象为石子个数，两两棋子中间的位置忽略，因为如果中间位置减小，后面的棋子可以移动相同的位置跟随，局面不会发生变化。

深入递归

  1. 一定要写出递推方案，在没有递推方案的情况下不要乱写程序浪费时间。

  2. 合法括号和非空子集都是改变状态的问题，合法括号有三种可选状态，加在左边，加在右边，加在两边；非空子集面临选或不选的问题，由此引出两条不同的递归分支，正好可以用二进制状态表示。

  3. 字符串全排列中使用交换法时每次交换包括自己，因为只有这样才能让自己的初始状态在递归树末端加入集合中。

  4. 逐步生成结果类问题还需要细微思考。

  5. 数独游戏中状态转移的方法是逐行逐单元格扫描。

  6. n皇后问题中只需要一位数组做记录，其中下标对应行数，元素值对应这一行放在了第几列，因为不能放在相同的行和列所以每行不受影响。检查对角线元素时利用行和列下标的关系直接判断，正对角线上的元素和列下标之和相等，副对角线元素列下标-行下标相等。
    因为每次检查只检查向上的几行，而向上几行已经将旧值覆盖，所以不需要回溯。

  7. 看清楚题意问的是什么，对于只要结果的可以不用保存所有数据并返回，直接打印即可；对于需要保存路径的用全局变量最方便。

  8. 困难的串中判断是否为困难串的方法：ABCD[E] [i] –> AB[CE][D i] –> [ABC][DE i]

动态规划

  1. 划船渡河问题有两种较优策略：一种是最快的两个过去+最快的回来+最慢的两个过去+第二快的回来；第二种是每次最快的带最慢的，在局部对两种方案比较，更优的计入结果。

  2. 区间调度问题的策略是优先选择结束时间更早的工作，才能够参加更多的工作。

  3. 区间选点问题也是优先选结束点，如果区间内要求了点数，则从结束点开始向前选择。

  4. 区间覆盖问题中每次选择起始点小于等于start并且end最大的那个区间，下一次将start更新为选择的end+1，按照这个局部最优策略得到全局最优策略。

  5. 部分背包问题中并不是直接按价值最高去拿，而是按照性价比排序更加合理。

  6. 贪心策略是动态规划的特殊情况，动态规划需要根据历史数据求出最终结果，贪心策略只需要使用上一步数据求出最终结果。

  7. 对于数字三角形问题来说，得到当前状态的结果只需要使用上一状态结果，因此不用保存所有历史状态，只需要保存上一状态即可。

  8. 一个字符串的 子序列 是指这样一个新的字符串：它是由原字符串在不改变字符的相对顺序的情况下删除某些字符（也可以不删除任何字符）后组成的新字符串。

  9. 部分背包问题是每种物品可以取走一部分，01背包是只能选或不选，完全背包是一类物品可以选任意多个。

  10. 最长上升子序列最优解法是在dp中维护一个值最小的递增子序列，扫描完所有数组元素的时候，dp中维护的Lis的最后下标就是最长递增子序列的长度。

线性结构

  1. 实现线性表的时候，多一个尾指针，能够在插入时无需遍历链表直接插入。

  2. 返回有环链表环的起始点一题中使用快慢指针原理，如果有环则快慢指针会相撞，相撞点距离环路起点的距离和头指针距离环路起点的距离相同。

  3. 维护最小值的栈一题中，需要申请一个辅助栈空间，维护当前元素下的最小值，可以有两种方案，一种是针对每个元素都有一个最小值在辅助栈中，出栈时一起出栈，这种方法辅助栈中的重复元素较多；还有一种是只维护当前情况下的最小值，出栈时只有最小值出栈的情况下辅助栈才会弹出。

哈希

  1. HashMap原理是利用数组+链表（拉链法）+hash映射实现的。

  2. HashSet持有HashMap的实例，只利用HashMap的key值存放数据，value值为空，其他功能直接调用HashMap的功能。

  3. 布隆过滤器原理是使用一个很长的二进制向量表+一系列散列函数将一个元素映射到向量表中的多个位置，通过判断向量表中的对应位置是否为1，得出元素是否在集合中。如果对应位置为0则元素一定不在，如果为1则一定在，具有一定的误判率。

  4. 一致性Hash没有实现。

树结构

  1. 树的层序遍历利用队列进行，弹出一个加入n个，为了实现遍历出层次效果，使用双指针记录层级关系，即游标指针p和指向每层最后一个元素的指针last，配合判断是否遍历完了一层。

  2. 树的高度是从叶子节点到根节点的最大值，树的深度是从根节点到叶子节点的最大值。

  3. 二叉查找树的删除：1.孩子节点之间删除2.有一个孩子，将孩子托付给父节点3.有两个孩子，从右侧子树中找一个最小节点顶替自己，然后递归删除这个最小节点，最终删除的情况一定在1和2情况中。

  4. 二叉查找树后继节点：1.如果有右子树一定为右子树中的最小值。2.如果没有右子树，向上找第一个是左孩子的节点的父节点。
    二叉查找树前驱节点：1.如果有左子树一定为左子树中的最大值。2.如果没有左子树，向上找第一个是右孩子的节点的父节点。

  5. AVL树是一颗自平衡的二叉查找树，只需要继承二叉查找树并且重写添加和删除方法保持平衡。重点在于添加或删除后的保持平衡，保持平衡的方法是左旋或者右旋。

  6. 红黑树也是一颗二叉查找树，在插入和删除时保持相对的平衡，但没有AVL树那样严格的平衡。

  7. 红黑树的插入和删除未掌握。

  8. 首个共同祖先解法二中返回值具有多个不同含义，未理解。

图结构

  1. 一定要切记图遍历的两大要素 访问标记和回溯。

  2. 拓扑排序中访问标识有三种状态，已访问，未访问，正在访问，利用正在访问标识可以判断有向图是否有环。

  3. 最小生成树中Kurskal算法的要素：Edge List + Union Find + 排序

  4. Dijkstra算法固定跑n次，每次从未访问节点中找一个距离最短的更新其邻居的距离，直到到所有点的距离更新完。

  5. 分酒问题的状态转移其实是bfs的过程，set判重就是bfs中的visited。

  6. 最重要的是对问题的抽象，把实际问题转移为抽象符号，抽象数据，然后用程序模拟问题的变化过程，也就是对实际问题建模。

  10. K倍区间：使用前缀和可以枚举区间和 两个任意区间和相减，所得结果为这两个区间和元素中间的区间和。
    a % m == k(k!=0) && b % m == k(k!=0) (a>b) ==> ( a - b ) % m == 0
    因此用两个相同余数的区间和相减一定为0，能够得出此区间为K倍区间，所以从n个相同余数的区间和中取出任意两个能够得到一个k倍区间。
    Cnm = n! / [(n-m)! * m!]
